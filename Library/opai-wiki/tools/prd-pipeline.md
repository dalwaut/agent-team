# PRD Pipeline
> Last updated: 2026-02-24 | Source: `tools/opai-prd/` | Agent: `scripts/prompt_prdgent.txt`

## Overview

PRD Pipeline is the product idea evaluation and project scaffolding system. Ideas flow in from the **mobile app** (or admin web tool), [YouTube videos](youtube-transcriber.md) (via Discord reaction or direct API call), CSV/Sheets import, or manual entry. Ideas are automatically evaluated by **PRDgent**, and — if scored highly — a full, written PRD is auto-generated by a second agent pass.

The core loop is:

```
Mobile/Admin submit → Supabase prd_ideas table → PRDgent scores idea
   → if verdict = good → PRDgent PRD agent generates full document
   → Human reviews in admin tool → Approve → Move to Project
```

## Architecture

```

Admin Browser (/prd/*)                             │
```

## PRDgent Agents

### Pass 1: Evaluation (`prompt_prdgent.txt`)

PRDgent evaluates raw ideas against 5 criteria and returns a structured JSON verdict.

| Criterion | Question |
|-----------|---------|
| Market Demand | Does a real, reachable audience need this? |
| Differentiation | What makes it meaningfully different? |
| Feasibility | Can this be built with reasonable effort and cost? |
| Monetization | Are there clear, defensible revenue paths? |
| Timing | Is the market ready for this now? |

**Verdict Rules:**

| Verdict | Condition |
|---------|-----------|
| `good` | Average ≥ 6.5 AND no single score below 4 |
| `not_ready` | Average 4–6.4, or one score below 4 |
| `poor` | Average below 4, or multiple scores below 4 |

### Pass 2: Full PRD Generation (`prompt_prdgent_prd.txt`)

Triggered automatically when verdict = `good`. Takes the idea data + evaluation JSON and generates a complete, professionally-written PRD with 12 sections:

1. Executive Summary
2. Problem Statement
3. Proposed Solution
4. Target Market & User Personas
5. Competitive Landscape (table with real competitors)
6. Feature Requirements (MVP + Phase 2 + Out of Scope)
7. Technical Architecture
8. Monetization Strategy (with pricing tiers + revenue projections)
9. Go-to-Market Strategy
10. Success Metrics (table with 3-month/6-month/year-1 targets)
11. Risks & Mitigations (table format)
12. Recommended Next Steps (sequenced action plan)

Output stored in `full_prd` column of `prd_ideas`. Used directly as `PRD.md` when idea is moved to a project folder.

## Data Model

Supabase table `prd_ideas`:

```sql
id            TEXT PRIMARY KEY  -- 'idea-' + 8 hex chars
name          TEXT NOT NULL
description   TEXT              -- full description / product overview
target_market TEXT
notes         TEXT
source        TEXT              -- manual | csv | mobile | json | sheets
status        TEXT              -- pending | evaluated | reviewed | approved | rejected | moved
pain_point    TEXT              -- mobile-origin (nullable for admin/CSV)
solution      TEXT              -- mobile-origin (nullable for admin/CSV)
evaluation    JSONB             -- PRDgent scores + verdict + analysis
full_prd      TEXT              -- Generated full PRD markdown (good verdicts)
submitted_by  UUID              -- auth.users ref (nullable)
submitted_at  TIMESTAMPTZ
evaluated_at  TIMESTAMPTZ
project_path  TEXT              -- e.g. "Projects/my-app"
moved_at      TIMESTAMPTZ
```

### Status Flow

```
pending → (PRDgent auto-eval on submit)
        → evaluated (good)    → approved → moved
        → reviewed (not_ready/poor) → can still be manually approved
        → rejected
```

## Auto-Evaluation Flow (Background Task)

On every `POST /api/submit`:

1. Idea written to Supabase with `status: pending`
2. Response sent immediately: `{idea_id, status: "queued"}`
3. Background task `_auto_evaluate_and_prd` fires:
   - Fetches idea from Supabase
   - Runs Pass 1: PRDgent evaluation (300s timeout)
   - Updates `evaluation`, `status`, `evaluated_at`
   - If `verdict == good`: runs Pass 2: full PRD generation (360s timeout)
   - Updates `full_prd`
4. Admin dashboard shows updated results

Total time for a good idea: ~3–6 minutes (eval + PRD generation)

## Typical Workflow

### Mobile-first (new flow)
1. User opens mobile app → Command tab → "PRD Pipeline" tile
2. Fills out form (title, pain, solution, market, notes)
3. Taps "Submit to PRDgent" → gets immediate confirmation
4. 2–3 minutes later: admin opens `/prd/` dashboard, sees evaluated idea
5. If `good`: full PRD already generated and stored
6. Admin reviews → Approve → Move to Project → `Projects/<slug>/PRD.md` contains full document

### Admin web (original flow)
1. Import CSV or add manually
2. Select all pending → Run PRDgent (manual trigger)
3. Review evaluations, approve/reject
4. Move to Project (uses generated full_prd if available, scaffold otherwise)

## Known Limitations / Gotchas

- **Supabase service key required** — server needs `SUPABASE_SERVICE_KEY` env var or all DB ops fail
- **Background task failure** — if eval crashes, idea stays `pending`. Admin can manually trigger eval from the web tool. A `[eval-error]` note is appended to the idea's notes field.
- **Full PRD timeout** — generation has a 360s timeout. Very complex ideas on slow machines may time out; use manual "Generate PRD" button to retry.
- **JSON extraction from claude output** — uses regex to find first `{...}` block containing `"results"` (eval) or `"prd"` (full PRD). Malformed JSON returns 502.
- **Batch evaluation limits** — manual `/api/evaluate` with large batches (>15 ideas) may time out; evaluate in batches of 5–10.
- **RLS get_my_role()** — admin policies use the shared `get_my_role()` security definer function. Ensure migration `017_fix_profiles_rls_recursion.sql` is applied first.
- **Move requires approved/evaluated** — `reviewed` ideas (not_ready/poor) need manual approval before moving.
