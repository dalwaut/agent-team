## Enhanced Knowledge: Python Testing Patterns [python-development/python-testing-patterns]
- Follow AAA pattern: Arrange (setup), Act (execute), Assert (verify) -- one behavior per test
- Use pytest fixtures with yield for setup/teardown; scope levels: function (default), module, session
- Parametrize tests with @pytest.mark.parametrize for multiple input/output pairs; use pytest.param(id="...") for readable names
- Mock external deps with unittest.mock: patch() for module-level, Mock/MagicMock for objects, side_effect for sequences/errors
- Test exceptions with pytest.raises(ExceptionType, match="pattern") -- always verify error messages
- Async tests: use @pytest.mark.asyncio; async fixtures with yield
- monkeypatch for env vars (setenv/delenv) and object attributes; tmp_path for temporary file operations
- Property-based testing with hypothesis: @given(st.text()), @given(st.integers()) to find edge cases
- Test naming: test_<unit>_<scenario>_<expected> (e.g., test_create_user_with_duplicate_email_raises_conflict)
- Coverage: pytest --cov=myapp --cov-fail-under=80 --cov-report=term-missing
- conftest.py for shared fixtures; organize tests/ into test_unit/, test_integration/, test_e2e/
- Test retry behavior with Mock side_effect=[Error, Error, {"ok": True}] and assert call_count

## Enhanced Knowledge: JavaScript Testing Patterns [javascript-typescript/javascript-testing-patterns]
- Framework setup: Jest with ts-jest preset or Vitest with vite-native config; coverage thresholds in config (80%+ recommended)
- Unit tests: describe/it blocks with AAA pattern; beforeEach for fresh instances; vi.clearAllMocks() between tests
- Async testing: async/await with expect().rejects.toThrow(); mock fetch with vi.fn().mockResolvedValueOnce()
- Mocking modules: vi.mock("module") with factory function; dependency injection via interfaces for testable services
- Spying: vi.spyOn(object, "method") to verify call counts and arguments; mockRestore() in afterEach
- React component tests with Testing Library: render, screen.getByRole/getByLabel/getByText, fireEvent; prefer semantic queries over data-testid
- Hook testing with renderHook + act() for state updates
- Test factories with faker: createUserFixture(overrides?) for consistent, randomized test data
- Snapshot testing: toMatchSnapshot() for UI components; update snapshots deliberately, not blindly
- Timer testing: vi.useFakeTimers(), vi.advanceTimersByTime(ms), vi.useRealTimers()
- Integration tests: supertest for API endpoints with real DB setup/teardown; truncate tables in beforeEach

## Enhanced Knowledge: E2E Testing Patterns [developer-essentials/e2e-testing-patterns]
- E2E tests: focus on critical user journeys (login, checkout, signup), not unit-level logic or edge cases
- Playwright config: fullyParallel, retries in CI, trace on-first-retry, screenshot only-on-failure, multi-browser projects
- Page Object Model: encapsulate page locators and actions in classes; use getByLabel/getByRole selectors, not CSS classes
- Waiting strategies: NEVER use fixed waitForTimeout; use auto-waiting assertions (expect().toBeVisible()), waitForURL, waitForResponse
- Network mocking: page.route() to mock/intercept API responses for error states, slow networks, third-party services
- Selectors: use data-testid or semantic selectors (getByRole, getByLabel); avoid CSS class selectors or nth-child
- Visual regression: toHaveScreenshot() with maxDiffPixels tolerance; capture default, hover, disabled states
- Accessibility in E2E: integrate @axe-core/playwright; expect(violations).toEqual([]) per page
- Tests must be independent, deterministic, and clean up their own data; use fixtures with setup/teardown
- Debug with --headed, --debug, page.pause(), test.step() for better reporting

## Enhanced Knowledge: Bats Testing Patterns [shell-scripting/bats-testing-patterns]
- Bats uses @test blocks with run command; check $status for exit codes, $output for stdout, ${lines[N]} for specific lines
- setup() runs before each test (create tmpdir, source script under test); teardown() cleans up (rm -rf $TMPDIR)
- setup_file()/teardown_file() for expensive one-time resources shared across all tests in a file
- Output assertions: exact match [ "$output" = "expected" ]; substring [[ "$output" == *"pattern"* ]]; regex [[ "$result" =~ ^[0-9]+$ ]]
- File assertions: [ -f "$path" ] for existence; diff for content comparison; [ "$(wc -c < file)" -eq N ] for size
- Mock external commands: create stub scripts in $STUBS_DIR prepended to PATH; or export mock functions with export -f
- Variable stubbing: export env vars in test, unset in alternate test; verify both override and default behavior
- Test error conditions: missing files, invalid input, permission denied; verify both exit code and error message content
- Use load test_helper to share assertion helpers (assert_file_exists, assert_file_equals) and common setup across test files
- Use skip for conditional tests (missing deps); test shell compatibility (bash, sh, dash) when scripts must be portable
- Run in CI: bats tests/*.bats --tap for TAP-compatible output; --parallel N for speed
