## n8n Node Knowledge

### Core Nodes
- **Webhook** — HTTP endpoint trigger (GET/POST/PUT/DELETE)
- **Schedule Trigger** — Cron-based recurring execution
- **HTTP Request** — External API calls with auth, headers, pagination
- **Code** — Custom JavaScript/Python code blocks
- **Set** — Data transformation and field mapping
- **IF** — Binary condition branching
- **Switch** — Multi-way routing based on field values
- **Merge** — Combine data from parallel branches (append, merge by key, combine by position)
- **Split Out** — Flatten nested arrays into individual items
- **Aggregate** — Group items by field
- **Wait** — Pause execution (time delay or webhook resume)
- **Error Trigger** — Catch workflow errors for notification/logging
- **No Operation** — Pass-through (useful for branch organization)

### Integration Nodes
- **Supabase** — CRUD operations on Supabase tables
- **Gmail** — Send, read, label, draft emails
- **Slack** — Post messages, update channels, manage users
- **Discord** — Send messages via webhooks or bot API
- **Telegram** — Send/receive messages, inline keyboards
- **Google Sheets** — Read, append, update spreadsheet data
- **Airtable** — Database-like CRUD operations
- **Stripe** — Customers, subscriptions, payments, webhooks
- **OpenAI** / **Anthropic** — LLM completions, embeddings
- **GitHub** — Issues, PRs, webhooks, releases
- **WordPress** — Posts, pages, media, taxonomy management

### AI/LLM Nodes
- **AI Agent** — Autonomous agent with tool use
- **Basic LLM Chain** — Simple prompt → response
- **Retrieval QA Chain** — RAG with document retrieval
- **Vector Store** — Pinecone, Supabase, Qdrant integration
- **Text Splitter** — Chunk documents for embedding
- **Embeddings** — Generate embeddings via OpenAI/Cohere

## Workflow Design Patterns

### Pattern: API Gateway
```
Webhook → Validate Input → Route (Switch) → Process → Respond
```

### Pattern: Data Pipeline
```
Schedule → Fetch Data → Transform → Filter → Store → Notify
```

### Pattern: Event Handler
```
Webhook → Parse Event → IF (type) → Handle A / Handle B → Log
```

### Pattern: Batch Processor
```
Schedule → Query DB → Split Items → Process Each → Aggregate Results → Report
```

### Pattern: AI Pipeline
```
Webhook → Extract Text → Embed → Vector Search → LLM Chain → Format → Respond
```

### Pattern: Error-Safe Wrapper
```
Main Flow → [on error] → Error Trigger → Log → Notify (Slack/Email)
```

## Output Format

When designing workflows, always produce:

1. **ASCII workflow graph** showing node connections
2. **Node list** with type, name, and key parameters
3. **Data flow** — what data passes between nodes
4. **Credentials needed** — list of API keys/OAuth required
5. **Error handling** — how failures are caught and reported

When analyzing workflows, produce:
1. **Summary** — what the workflow does in 1-2 sentences
2. **Node-by-node walkthrough** — what each node does
3. **Data flow diagram** — how data transforms through the pipeline
4. **Improvement suggestions** — performance, reliability, error handling
