- State categories: Local (useState/useReducer), Global (Redux Toolkit, Zustand, Jotai), Server (React Query/SWR/RTK Query), URL (React Router), Form (React Hook Form)
- Selection criteria: small app -> Zustand or Jotai; large app -> Redux Toolkit; heavy server interaction -> React Query + light client state; atomic updates -> Jotai

- React Query: query key factories (`userKeys.detail(id)`), staleTime/gcTime configuration, optimistic mutations with rollback (onMutate/onError/onSettled), invalidation on mutation success
- Best combination: Zustand for client/UI state (sidebar, modal, theme) + React Query for server state (users, products, stats)
- Do's: colocate state, use selectors, normalize data, full TypeScript, separate concerns
- Don'ts: over-globalize, duplicate server state, mutate directly, store derived data, mix paradigms

## Enhanced Knowledge: Responsive Design [ui-design/responsive-design]
- Mobile-first breakpoints: base (<640px), sm (640px), md (768px), lg (1024px), xl (1280px), 2xl (1536px) -- Tailwind equivalents
- Container queries: `container-type: inline-size`, `@container card (min-width: 400px)`, container query units (cqi, cqw); Tailwind: @container, @md:, @lg:
- Fluid typography: clamp() with viewport-relative scaling, e.g., `font-size: clamp(1rem, 0.9rem + 0.5vw, 1.125rem)`; define --text-xs through --text-4xl as fluid custom properties
- CSS Grid responsive: `grid-template-columns: repeat(auto-fit, minmax(min(300px, 100%), 1fr))` for automatic wrapping; named grid areas with breakpoint overrides
- Responsive navigation: mobile hamburger (hidden lg:flex), slide-down menu, desktop horizontal nav; use aria-expanded and aria-controls
- Responsive images: picture element for art direction, srcset+sizes for resolution switching, loading="lazy" for offscreen images
- Responsive tables: horizontal scroll wrapper (overflow-x-auto, min-w-[600px]) or card-based layout for mobile (hidden md:table + md:hidden cards)
- Viewport units: use 100dvh instead of 100vh on mobile (accounts for browser UI); 100svh for minimum viewport
- Best practices: content-based breakpoints, fluid over fixed values, 44x44px touch targets, logical properties for i18n, test on real devices
