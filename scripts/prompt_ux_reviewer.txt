## Enhanced Knowledge: Design System Patterns [ui-design/design-system-patterns]
- Token hierarchy: Layer 1 Primitive (raw values: --color-blue-500, --space-4) -> Layer 2 Semantic (meaning: --text-primary, --surface-default) -> Layer 3 Component (usage: --button-bg, --card-border)
- Name tokens by purpose (text-primary) not visual description (dark-gray); treat token changes as API changes with semver
- Theme switching: CSS custom properties per theme class (.light/.dark on :root); detect system preference via prefers-color-scheme media query; persist choice to localStorage
- Variant system: use class-variance-authority (CVA) for variant+size combinations with defaultVariants; compose with cn() utility
- Style Dictionary for multi-platform token generation: CSS variables, SCSS, iOS Swift, Android XML from single JSON source
- Component architecture patterns: compound components, polymorphic (as prop), headless UI, slot-based composition
- Common issues: token sprawl without clear hierarchy, mixed naming conventions, hardcoded values instead of tokens, missing dark mode adaptation, circular token references

## Enhanced Knowledge: Responsive Design [ui-design/responsive-design]
- Mobile-first breakpoints: base (mobile) -> 640px (sm) -> 768px (md) -> 1024px (lg) -> 1280px (xl) -> 1536px (2xl)
- Container queries for component-level responsiveness: container-type: inline-size; @container (min-width: 400px) -- independent of viewport
- Fluid typography with clamp(): --text-base: clamp(1rem, 0.9rem + 0.5vw, 1.125rem); same pattern for spacing scale
- CSS Grid auto-fit: grid-template-columns: repeat(auto-fit, minmax(min(300px, 100%), 1fr)) for automatic wrapping
- Grid named areas: define grid-template-areas per breakpoint for page layouts (header/main/sidebar/footer)
- Responsive images: use picture element with source media for art direction; srcset + sizes for resolution switching; lazy loading for off-screen
- Responsive tables: horizontal scroll wrapper with min-width on desktop; card-based layout on mobile (hidden md:table + md:hidden cards)
- Use dvh (dynamic viewport height) instead of vh on mobile to account for browser UI
- Touch targets: 44x44px minimum on mobile; test on real devices, not just simulators
- Common issues: horizontal overflow from fixed widths, 100vh on mobile, font size too small, z-index stacking context breaks

## Enhanced Knowledge: Interaction Design [ui-design/interaction-design]
- Motion should communicate (feedback, orientation, focus, continuity), not just decorate
- Timing: 100-150ms for micro-feedback (hovers, clicks); 200-300ms for small transitions (toggles, dropdowns); 300-500ms for medium (modals, pages)
- Easing: ease-out for entering elements; ease-in for exiting; spring for playful overshoot
- Loading states: skeleton screens to preserve layout shape; determinate progress bars with smooth animation; distinguish loading vs empty
- State transitions: smooth toggle animations with spring physics; page transitions with AnimatePresence (fade+slide)
- Feedback patterns: ripple effects on click, scale on hover (1.02) and tap (0.98), confirmation animations
- Always respect prefers-reduced-motion: set animation-duration and transition-duration to near-zero in reduce-motion media query
- Performance: only animate transform and opacity for 60fps; avoid animating width, height, top, left
- Gesture interactions: swipe-to-dismiss with drag constraints and threshold; gesture alternatives for keyboard users
- Common issues: janky animations from non-composited properties, over-animation causing fatigue, blocking user input during animations, memory leaks from uncleaned animation listeners
