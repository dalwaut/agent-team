AUDIT SCOPE:

1. **Insecure Token & Credential Storage**

   Scan all JS/TS files in the mobile project:
   - Flag `AsyncStorage.setItem()` storing auth tokens, JWTs, passwords, or session data — AsyncStorage is unencrypted plaintext. Require `expo-secure-store` for sensitive values.
   - Flag tokens or API keys stored in `useState` or module-level variables that are logged
   - Flag credentials passed as URL params (`?token=...`) — appear in logs, proxies, browser history
   - Flag auth tokens stored in Redux state without persistence encryption

2. **Hardcoded Secrets in Bundle**

   Scan all source files and config:
   - Flag hardcoded Supabase anon keys, API keys, or service URLs in source code (not environment config)
   - Flag API base URLs hardcoded as string literals — should come from `app.config.js` or `expo-constants`
   - Flag `__DEV__` blocks that include real API keys for "dev convenience"
   - Flag `.env` values being bundled via `babel-plugin-transform-inline-environment-variables` without review of what's included

3. **Network Security**

   Scan network call patterns:
   - Flag HTTP (not HTTPS) endpoints in any API calls
   - Flag `fetch()` calls without timeout — can hang indefinitely
   - Flag self-signed certificate acceptance or disabled SSL verification
   - Flag missing `Authorization` header validation on authenticated endpoints (cross-reference with docs/mobile-api-reference.md)
   - Flag Supabase realtime subscriptions that don't unsubscribe on component unmount — connection leak

4. **Deep Link & Navigation Security**

   Scan navigation config and app scheme:
   - Flag unvalidated deep link parameters passed directly to navigation state or API calls
   - Flag deep links that can trigger authenticated actions without re-verification
   - Flag `Linking.addEventListener` handlers that don't sanitize the incoming URL
   - Check `app.json`/`app.config.js` for scheme registration — note if universal links (HTTPS) are not configured (custom schemes like `myapp://` are spoofable)

5. **Expo & Build Security**

   Scan build config and EAS configuration:
   - Flag Expo SDK version — if more than 2 major versions behind current (SDK 52 as of early 2026), flag as High
   - Flag `expo-updates` configured without code signing (`codeSigningCertificate`) — OTA updates without verification
   - Flag `eas.json` with secrets referenced as plain environment variables that could be logged in CI output
   - Flag `app.json` `extra` fields containing secrets (bundled into the app binary)
   - Flag missing `android.permissions` restriction — overly broad permissions increase attack surface

6. **Data Exposure**

   Scan logging and error handling:
   - Flag `console.log()` statements that include auth tokens, user PII, or API responses with sensitive data
   - Flag error boundaries that display raw error messages (including stack traces) to users
   - Flag Sentry/crash reporting SDK configured without PII scrubbing (data before send hook)
   - Flag clipboard operations that copy sensitive data (tokens, passwords) — no clear-on-background mechanism

7. **API Contract Compliance**

   Cross-reference mobile API calls against docs/mobile-api-reference.md:
   - Flag any API calls to undocumented endpoints (could be deprecated/removed)
   - Flag missing error handling for 401/403 responses — should trigger re-auth flow, not silent failure
   - Flag API calls that don't send the `Authorization: Bearer <token>` header on protected routes

## Enhanced Knowledge: React Native Architecture [frontend-mobile-development/react-native-architecture]
- Project structure: `app/` (Expo Router screens with groups like `(auth)/`, `(tabs)/`), `components/` (ui + features), `hooks/`, `services/`, `stores/`, `utils/`, `types/`
- Auth flow: use `expo-secure-store` for tokens (never AsyncStorage); protect routes via `useSegments()` + redirect logic in AuthProvider; check auth on mount
- Offline-first with React Query: `PersistQueryClientProvider` + `AsyncStorage` persister; sync online status via NetInfo; use `networkMode: 'offlineFirst'`; optimistic updates with `onMutate`/`onError` rollback
- Performance: use `FlashList` over FlatList for long lists; `memo` + `useCallback` for list items; `estimatedItemSize` required; `removeClippedSubviews` + `maxToRenderPerBatch` + `windowSize` tuning
- Native modules: wrap platform checks (`Platform.OS !== 'web'`) for haptics, biometrics, notifications; register push tokens via `expo-notifications` with projectId
- Platform-specific code: use `Platform.select()` for shadow (iOS) vs elevation (Android); platform-specific files (`.ios.tsx`, `.android.tsx`) for divergent implementations
- Navigation: Expo Router file-based routing; `router.push()`, `router.replace()`, `router.back()`; dynamic routes with `[id].tsx` + `useLocalSearchParams`
- EAS Build: development (simulator), preview (internal APK), production (auto-increment); OTA updates with `eas update --branch production`
- Do: use StyleSheet.create (never inline styles), test on real devices, use Reanimated for 60fps animations, use error boundaries
- Don't: fetch in render, ignore platform differences, store secrets in code, skip error boundaries

## Enhanced Knowledge: React Native Design [ui-design/react-native-design]
- StyleSheet: always use `StyleSheet.create` for performance; dynamic styles via array syntax `[styles.base, condition && styles.variant]`
- Flexbox: `flexDirection: 'column'` (default), `'row'` for horizontal; use `gap` property; `justifyContent` for main axis, `alignItems` for cross axis
- Platform-specific styling: `Platform.select({ ios: { shadowColor/Offset/Opacity/Radius }, android: { elevation } })`; font families differ per platform
- React Navigation: type-safe with `ParamList` types for all navigators; Stack, Tab, Drawer navigators; customize `screenOptions` for consistent theming
- Reanimated 3: `useSharedValue` + `useAnimatedStyle` for 60fps animations; `withSpring`/`withTiming` for transitions; animations run on UI thread (worklets)
- Gesture Handler: `Gesture.Pan()` with `onUpdate`/`onEnd` for drag; `GestureDetector` wrapper; combine with Reanimated for gesture-driven animations
- Press feedback: `AnimatedPressable` with scale animation (`withSpring(0.95)` on press in, `withSpring(1)` on press out) + haptic feedback
- Safe areas: always use `SafeAreaView` or `useSafeAreaInsets`; test on notched devices (iPhone, Android cutouts)
- Common issues: gesture conflicts (use `simultaneousHandlers`), navigation type errors (define ParamList types), animation jank (move to UI thread), memory leaks (cleanup in useEffect)
- Never use ScrollView with `.map()` for long lists — use FlatList/FlashList; memoize list items with `React.memo` + `useCallback`
