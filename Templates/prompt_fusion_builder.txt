## SHORTCODE ARCHITECTURE

Every Fusion Builder page follows this strict nesting hierarchy:

```
[fusion_builder_container] ← Section wrapper (background, padding, layout)
  [fusion_builder_row]     ← Row wrapper (always required inside container)
    [fusion_builder_column] ← Column (defines width, spacing, animation)
      [element shortcodes]  ← Content elements (text, buttons, images, etc.)
    [/fusion_builder_column]
  [/fusion_builder_row]
[/fusion_builder_container]
```

Rules:
- Every page MUST start with `[fusion_builder_container]` and end with `[/fusion_builder_container]`
- Every container MUST contain exactly one `[fusion_builder_row]`
- Every row MUST contain one or more `[fusion_builder_column]` elements
- Column widths in a row must sum to 1 (e.g., 1_2 + 1_2, or 1_3 + 1_3 + 1_3, or 1_1)
- Elements go INSIDE columns only, never directly in rows or containers

## CONTAINER PARAMETERS

```
[fusion_builder_container
  type=""                        ← "flex" (modern) or "legacy" (old). Use "flex" for new pages
  hundred_percent="yes|no"       ← Full-width container (edge to edge)
  hundred_percent_height="yes|no" ← Full viewport height
  background_color="#hex"        ← Solid background color
  background_image="URL"         ← Background image URL
  background_position="center center" ← Image position
  background_repeat="no-repeat"  ← Repeat behavior
  background_parallax="none|fixed|up|down" ← Parallax effect
  overlay_color="#hex"           ← Color overlay on background image
  overlay_opacity="0.0-1.0"     ← Overlay transparency
  padding_top="60px"            ← Top padding (use px, %, or em)
  padding_bottom="60px"         ← Bottom padding
  padding_left="30px"           ← Left padding
  padding_right="30px"          ← Right padding
  margin_top="0px"              ← Top margin
  margin_bottom="0px"           ← Bottom margin
  border_color="#hex"            ← Border color
  border_size="1px"             ← Border width
  equal_height_columns="yes|no" ← Force equal column heights
  flex_align_items="stretch|flex-start|center|flex-end" ← Vertical alignment (flex only)
  flex_justify_content="flex-start|center|flex-end|space-between|space-around"
  admin_label="Section Name"     ← Label in backend editor (organizational)
  id=""                          ← HTML id attribute
  class=""                       ← HTML class attribute
  hide_on_mobile="small-visibility,medium-visibility,large-visibility"
]
```

## BRAND COLOR APPLICATION

Read design-specs.md for the project's specific brand colors. Apply them consistently:

- **Hero sections**: Brand background or full-bleed image with dark overlay + white text
- **Alternating sections**: Alternate between light background and white
- **CTA sections**: Primary accent color background with contrasting text
- **Buttons**: Primary accent background, white text. Hover → slightly darker
- **Borders/dividers**: Use the designated border/accent color
- **Text**: Use the designated dark text color for body and headings

## OUTPUT RULES

1. Output ONLY valid Fusion Builder shortcode — no explanatory text mixed in
2. Every container must have proper open and close tags
3. Column widths must sum to 1 per row
4. Use `type="flex"` on containers for modern layout
5. Include `admin_label` on containers for backend organization
6. Apply brand colors from design-specs.md consistently
7. Use consistent spacing: `padding_top="60px" padding_bottom="60px"` for sections
8. Add animations sparingly — `fade` at `0.5` speed is elegant and subtle
9. Include responsive visibility where appropriate
10. Use semantic HTML inside `fusion_text` blocks (proper heading levels, lists, links)
11. Include placeholder text like `[IMAGE_PLACEHOLDER]` where product/hero images should go
12. All shortcode must be complete — no truncated or partial blocks

For each page you build, provide:
- The complete Fusion Builder shortcode (ready to paste)
- A brief section map listing each container and its purpose
- Notes on any placeholder content that needs real data

Format your response as valid markdown with the shortcode in code blocks.
Do not ask for clarification.
